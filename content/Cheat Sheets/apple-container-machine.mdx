## What It Is

`container machine` gives you a persistent Linux environment on your Mac. Unlike an app-style container, a container machine is meant to feel like a lightweight Linux workstation with your macOS home directory mapped in.

```bash
# Your macOS home directory is available inside the machine
container machine run -n dev pwd
```

---

## Installation & Setup

### Start the Apple Container Service First

```bash
container system start
container system status
```

### Useful Aliases

```bash
alias cm='container machine'
alias cmls='container machine ls'
alias cmrun='container machine run'
```

### Quickstart

```bash
container machine create alpine:latest --name dev
container machine run -n dev whoami
container machine run -n dev pwd
container machine run -n dev
```

## Machine Lifecycle

### Create Machines

```bash
container machine create alpine:latest --name dev
container machine create ubuntu:latest --name ubuntu-dev --cpus 4 --memory 8G
container machine create --set-default alpine:latest --name default-dev
container machine create --no-boot alpine:latest --name cold-dev
container machine create --home-mount ro alpine:latest --name safe-dev
```

### Open a Shell or Run One Command

```bash
container machine run -n dev
container machine run -n dev uname -a
container machine run -n dev -- cat /etc/os-release
container machine run -n dev --root whoami
container machine run -n dev -e APP_ENV=dev -- sh -lc 'env | grep APP_ENV'
```

### List, Inspect, Stop, Delete

```bash
container machine ls
container machine ls --format json
container machine inspect dev
container machine stop dev
container machine rm dev
```

### Default Machine

```bash
container machine set-default dev
container machine run
container machine inspect
container machine stop
```

## Configuration & Resources

### Change CPUs, Memory, Home Mount

```bash
container machine set -n dev cpus=4 memory=8G
container machine set -n dev home-mount=ro
container machine set -n dev home-mount=none

# Changes apply after restart
container machine stop dev
container machine run -n dev -- nproc
```

### Custom Kernel & Nested Virtualization

```bash
# Requires Apple Silicon M3+, macOS 15+, and a kernel with CONFIG_KVM=y
container machine create \
  --virtualization \
  --kernel /path/to/vmlinux-kvm \
  --name kvm-dev \
  alpine:latest

container machine set -n dev virtualization=true kernel=/path/to/vmlinux-kvm
container machine stop dev
container machine run -n dev -- ls -l /dev/kvm

# Reset to the default kernel
container machine set -n dev kernel=
```

## Common Workflows

### Daily Linux Dev Environment

```bash
container machine create ubuntu:latest --name work --set-default
container machine run
cd ~/Code/my-project
```

### Run Package or Build Commands

```bash
container machine run -n work -- apt update
container machine run -n work -- apt install -y build-essential git
container machine run -n work -- make test
```

### Work as Root When Needed

```bash
container machine run -n work --root apt update
container machine run -n work --root id
```

### Test Across Distros

```bash
container machine create alpine:latest --name alpine-dev
container machine create ubuntu:latest --name ubuntu-dev
container machine run -n alpine-dev -- cat /etc/os-release
container machine run -n ubuntu-dev -- cat /etc/os-release
```

## Build a Custom Machine Image

### Minimal Ubuntu Machine Image

```dockerfile
FROM ubuntu:24.04

ENV container container

RUN apt-get update && \
    apt-get install -y \
    dbus systemd openssh-server net-tools iproute2 iputils-ping curl wget vim-tiny man sudo && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/* && \
    yes | unminimize

RUN >/etc/machine-id
RUN >/var/lib/dbus/machine-id

RUN systemctl set-default multi-user.target
```

### Build and Create a Machine from It

```bash
container build -t local/ubuntu-machine:latest .
container machine create local/ubuntu-machine:latest --name ubuntu-machine
container machine run -n ubuntu-machine
```

### Custom First-Boot User Setup

```bash
# Add an executable script to the image at:
/etc/machine/create-user.sh
```
